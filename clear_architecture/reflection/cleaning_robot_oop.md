# Рефлексия по ../cleaning_robot_oop.py

Задание состояло в написании программы для дистанционного управления роботом.
Реализацию нужно было выполнить с помощью ООП.
Сравниваю текущее решение с решением в процедурном стиле.

## Преимущества решения
- вся логика и данные инкапсулированы внутри класса
- улучшена валидация, уменьшена повторяемость кода
- обработка ошибок вынесена в отдельную часть

## Недостатки решения

- _transfer_to_cleaner удобнее передавать через интерфейс-параметр в конструктор, а не писать внутри класса. Это позволяет легко адаптировать программу для разных роботов, меняя только передаваемую функцию.
