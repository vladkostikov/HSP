# Рефлексия по ../cleaning_robot.py

Задание состояло в написании программы для дистанционного управления роботом.
После выполнения задания стал доступен пример решения в стиле "спагетти-код".

## Сравнение подходов

Я выбрал более сложный, но лучше поддерживаемый подход.
Реализовал класс `CleaningRobot`, который выполняет всю работу через единственный публичный метод `.call()`, обрабатывающий все внешние запросы.

Внутренние методы разделены на три группы:
- обработка команд
- действия робота
- форматирование вывода

## Преимущества решения

- вся логика инкапсулирована внутри класса
- отсутствуют if-else ветвления
- нет глобальных переменных
- простое добавление новых методов

## Недостатки решения

- ограниченная реализация поворотов (только кратные 90°)
- валидация смешана с основной логикой в хендлерах
- множественные return внутри методов
